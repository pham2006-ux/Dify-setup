# 要件定義書：議事録自動要約ボット

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | 議事録自動要約ボット |
| **Difyアプリ種別** | チャットフロー（advanced-chat） |
| **対象ユーザー** | 全社員（会議参加者・議事録担当者） |
| **解決する業務課題** | 会議メモ・録音テキストから議事録を手作業で構造化する工数（1会議あたり15〜30分） |

## 2. 機能要件

### 2.1 基本機能
- ユーザーが会議メモや文字起こしテキストを貼り付けると、構造化された議事録を自動生成する
- 会話メモリにより、追加の修正依頼やフォローアップ質問に対応する

### 2.2 出力フォーマット
議事録は以下の構成で出力する：

```
📋 議事録
━━━━━━━━━━━━━━━━━━━━━━

■ 会議基本情報
- 会議名：（テキストから推定）
- 日時：（テキストから推定、不明な場合は「要記入」）
- 参加者：（テキストから推定）

■ 議題と議論内容
1. [議題1]
   - 議論ポイント
   - 発言者（判別可能な場合）

■ 決定事項
- [ ] 決定事項1
- [ ] 決定事項2

■ アクションアイテム
| 担当者 | タスク | 期限 |
|--------|--------|------|
| ○○さん | タスク内容 | 期限 |

■ 次回予定
- 次回会議の日時・議題（言及がある場合）

■ 備考
- 保留事項や注意点
```

### 2.3 対応パターン
1. **詳細な文字起こし** → フルフォーマットの議事録を生成
2. **箇条書きメモ** → メモを補完して議事録フォーマットに整形
3. **追加修正** → メモリにより「決定事項を追加して」等の修正指示に対応

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| **正確性** | テキストに書かれていない内容を捏造しない |
| **言語** | 日本語で出力 |
| **入力上限** | LLMのコンテキスト上限まで（通常の会議メモは問題なし） |
| **メモリ** | 有効（直近10ターン保持） |

## 4. ワークフロー構成

```
開始（ユーザー入力）→ LLM（議事録生成）→ 回答
```

### ノード詳細

| ノード | 種別 | 役割 |
|--------|------|------|
| 開始 | start | ユーザーの会議メモ/テキストを受け取る |
| LLM | llm | 構造化された議事録を生成する |
| 回答 | answer | 生成された議事録をユーザーに表示する |

## 5. モデル設定

| 項目 | 設定値 |
|------|--------|
| **モデル** | gemini-2.5-flash |
| **Temperature** | 0.3（正確性重視） |
| **メモリ** | 有効 / ウィンドウサイズ10 |

## 6. 想定される利用シーン

1. 会議直後に文字起こしテキストを貼り付けて議事録を自動作成
2. 手書きメモをスマホで入力して構造化
3. 生成された議事録に「○○の決定事項を追記して」と修正指示
