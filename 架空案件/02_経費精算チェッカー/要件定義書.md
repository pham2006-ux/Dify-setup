# 要件定義書：経費精算チェッカー

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | 経費精算チェッカー |
| **Difyアプリ種別** | ワークフロー（workflow） |
| **対象ユーザー** | 経費申請を行う全社員 / 経理部の承認担当 |
| **解決する業務課題** | 経費申請の社内規程違反を経理部が1件ずつ目視チェックする工数（月間100件以上） |

## 2. 機能要件

### 2.1 入力項目（開始ノードの変数）
| 変数名 | 型 | 説明 | 例 |
|--------|------|------|------|
| expense_type | テキスト | 費用項目 | 交通費、交際費、消耗品費、出張旅費 |
| amount | 数値 | 金額（税込） | 15000 |
| date | テキスト | 利用日 | 2026-02-28 |
| description | テキスト | 利用内容の説明 | 顧客A社との会食（4名） |
| receipt | テキスト | 領収書の有無 | あり / なし |

### 2.2 経費規程ルール（コードノードに実装）
| ルール | 条件 | 判定 |
|--------|------|------|
| 交際費上限 | 1人あたり5,000円超（税込） | ⚠️ 要注意 |
| 交際費上限（役員） | 1人あたり10,000円超 | ⚠️ 要注意 |
| 消耗品上限 | 1件30,000円超は資産計上 | ❌ 不可 |
| 領収書必須 | 3,000円以上で領収書なし | ❌ 不可 |
| 出張旅費 | 100,000円超は事前申請必須 | ⚠️ 確認 |
| 日付チェック | 申請日から3ヶ月以上前 | ⚠️ 要注意 |

### 2.3 出力内容（LLMノードで生成）
- **判定結果**: ✅承認推奨 / ⚠️要確認 / ❌差戻し推奨
- **チェック詳細**: 各ルールの適合/違反状況
- **補足コメント**: 必要な追加書類や修正点の案内

## 3. ワークフロー構成

```
開始（経費情報入力）→ LLM（情報構造化）→ コード（規程チェック）→ If-Else（判定分岐）→ 回答
```

### ノード詳細

| ノード | 種別 | 役割 |
|--------|------|------|
| 開始 | start | 経費情報（項目・金額・日付・内容・領収書有無）を受け取る |
| LLM | llm | 入力テキストから経費情報をJSON形式に構造化する |
| コード | code | 社内規程ルールに基づくチェックを実行し、判定結果を出力 |
| If-Else | if-else | 判定結果（承認/要確認/差戻し）で回答を分岐 |
| 回答（承認） | answer | 承認推奨の結果とチェック詳細を表示 |
| 回答（差戻し） | answer | 差戻し理由と修正ガイダンスを表示 |

## 4. モデル設定

| 項目 | 設定値 |
|------|--------|
| **モデル** | gemini-2.5-flash |
| **Temperature** | 0.1（正確性最重視） |
| **メモリ** | 無効（ワークフローのため） |
