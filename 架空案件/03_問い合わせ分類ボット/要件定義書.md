# 要件定義書：顧客問い合わせ分類・Slack通知ボット

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | 顧客問い合わせ分類・Slack通知ボット |
| **Difyアプリ種別** | チャットフロー（advanced-chat） |
| **対象ユーザー** | カスタマーサポート担当者 |
| **解決する業務課題** | 問い合わせメールの手動分類・振り分け作業（1日50件以上）と緊急案件の対応遅延 |

## 2. 機能要件

### 2.1 分類カテゴリ
| カテゴリ | Slackチャンネル | 説明 |
|----------|----------------|------|
| 技術サポート | #support-tech | システム障害、バグ報告、操作方法 |
| 請求・料金 | #support-billing | 請求内容、返金、プラン変更 |
| 契約・法務 | #support-legal | 契約変更、解約、利用規約 |
| 一般問い合わせ | #support-general | その他全般 |

### 2.2 緊急度判定
| レベル | 条件 | 対応 |
|--------|------|------|
| 🔴 緊急 | システム停止、データ消失、セキュリティ | Slack通知 + @channel メンション |
| 🟡 通常 | 一般的な問い合わせ | Slack通知のみ |

### 2.3 出力内容
- カテゴリ分類結果
- 緊急度判定
- 問い合わせ要約（3行以内）
- 推奨対応アクション
- Slack自動通知（該当チャンネルへ）

## 3. ワークフロー構成

```
開始 → LLM（分類・緊急度判定）→ コード（部署・URL判定）→ If-Else（緊急度分岐）
  ├→ [緊急] HTTPリクエスト（Slack @channel通知）→ 回答
  └→ [通常] HTTPリクエスト（Slack通常通知）→ 回答
```

## 4. モデル設定

| 項目 | 設定値 |
|------|--------|
| **モデル** | gemini-2.5-flash |
| **Temperature** | 0.2 |
| **メモリ** | 有効 / ウィンドウサイズ10 |
