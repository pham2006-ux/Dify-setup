# 要件定義書：日報・週報作成支援ボット

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | 日報・週報作成支援ボット |
| **Difyアプリ種別** | チャットフロー（advanced-chat） |
| **対象ユーザー** | 日報・週報を提出する全社員 |
| **解決する業務課題** | 日報/週報を社内フォーマットに合わせて毎日書く手間（1日15〜20分） |

## 2. 機能要件

### 2.1 入力
- 箇条書きの業務メモ（今日やったこと、困ったこと、明日の予定等）
- 「日報」か「週報」の指定

### 2.2 出力テンプレート

#### 日報テンプレート
```
【日報】YYYY年MM月DD日（曜日）
所属：○○部  氏名：要記入

■ 本日の業務内容
1. [業務1]（作業時間：X時間）
   - 詳細・進捗
2. [業務2]（作業時間：X時間）
   - 詳細・進捗

■ 課題・懸念事項
- 課題内容

■ 明日の予定
- 予定内容

■ 連絡・共有事項
- （あれば記載）
```

### 2.3 テンプレート変換ノード
- LLMが構造化したデータをJinja2テンプレートで最終フォーマットに変換

## 3. ワークフロー構成

```
開始 → LLM（メモ→構造化データ）→ テンプレート変換（フォーマット適用）→ 回答
```

## 4. モデル設定

| 項目 | 設定値 |
|------|--------|
| **モデル** | gemini-2.5-flash |
| **Temperature** | 0.3 |
| **メモリ** | 有効 / ウィンドウサイズ10 |
