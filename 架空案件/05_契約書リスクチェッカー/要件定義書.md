# 要件定義書：契約書リスクチェッカー

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | 契約書リスクチェッカー |
| **Difyアプリ種別** | チャットフロー（advanced-chat） |
| **対象ユーザー** | 法務部・営業部（契約書レビュー担当） |
| **解決する業務課題** | 契約書のリスク条項を1件ずつ目視でチェックする工数。高リスク案件の見落とし防止 |

## 2. 機能要件

### 2.1 チェック対象リスク条項
| カテゴリ | チェック項目 |
|----------|------------|
| 金銭リスク | 違約金条項、損害賠償の上限、支払条件 |
| 期間リスク | 自動更新条項、解約通知期間、契約期間 |
| 競業リスク | 競業避止義務、秘密保持の範囲 |
| 責任リスク | 免責条項、瑕疵担保、知的財産権の帰属 |
| 法的リスク | 準拠法、管轄裁判所、反社条項 |

### 2.2 リスクレベル判定
| レベル | スコア | アクション |
|--------|--------|-----------|
| 🔴 高リスク | 70〜100 | Slackで法務部に緊急通知 + 詳細レビュー推奨 |
| 🟡 中リスク | 40〜69 | 注意事項付きで承認可能 |
| 🟢 低リスク | 0〜39 | 承認推奨 |

### 2.3 高リスク時のSlack通知（HTTPリクエスト）
- 高リスク判定時に法務部のSlackチャンネルへ自動通知
- 通知内容：リスクスコア、検出されたリスク条項、推奨アクション

## 3. ワークフロー構成

```
開始 → LLM（リスク条項抽出）→ コード（スコアリング）→ If-Else（リスクレベル分岐）
  ├→ [高リスク] HTTPリクエスト（Slack通知）→ 回答
  └→ [中・低リスク] 回答
```

## 4. モデル設定

| 項目 | 設定値 |
|------|--------|
| **モデル** | gemini-2.5-flash |
| **Temperature** | 0.2（正確性重視） |
| **メモリ** | 有効 / ウィンドウサイズ10 |
