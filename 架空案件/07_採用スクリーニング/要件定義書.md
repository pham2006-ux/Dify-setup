# 要件定義書：採用スクリーニングアシスタント

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | 採用スクリーニングアシスタント |
| **Difyアプリ種別** | ワークフロー（workflow） |
| **対象ユーザー** | 人事部・採用担当者 |
| **解決する業務課題** | 大量の応募書類（月間50件以上）の初期スクリーニングを手動で実施する工数 |

## 2. 機能要件

### 2.1 入力項目（開始ノードの変数）
| 変数名 | 型 | 説明 |
|--------|------|------|
| position | テキスト | 募集職種名 |
| requirements | テキスト | 必須スキル・経験（箇条書き） |
| candidate_info | テキスト | 候補者の経歴・スキル情報 |

### 2.2 評価基準
| 評価項目 | 配点 |
|----------|------|
| 必須スキル適合度 | 40点 |
| 経験年数 | 20点 |
| 関連業界経験 | 20点 |
| その他加点要素 | 20点 |

### 2.3 判定結果
| スコア | 判定 | アクション |
|--------|------|-----------|
| 80〜100 | ✅ 推奨 | 面接候補としてスプレッドシートに記録 |
| 50〜79 | 🟡 検討 | 追加検討としてスプレッドシートに記録 |
| 0〜49 | ❌ 不適合 | 不適合としてスプレッドシートに記録 |

### 2.4 HTTPリクエスト（Google Apps Script連携）
- 評価結果をGoogle スプレッドシートに自動記録
- GASのWebアプリURLに結果をPOST

## 3. ワークフロー構成

```
開始（候補者情報入力）→ LLM（評価）→ コード（スコアリング）→ If-Else → HTTPリクエスト（GAS記録）→ 回答
```

## 4. モデル設定

| 項目 | 設定値 |
|------|--------|
| **モデル** | gemini-2.5-flash |
| **Temperature** | 0.2 |
| **メモリ** | 無効（ワークフローのため） |
