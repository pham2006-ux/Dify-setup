# 要件定義書：見積書作成支援ボット

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | 見積書作成支援ボット |
| **Difyアプリ種別** | チャットフロー（advanced-chat） |
| **対象ユーザー** | 営業部・プリセールス |
| **解決する業務課題** | 要件ヒアリング内容から見積書ドラフトを手作業で作成する工数（1件30分〜1時間） |

## 2. 機能要件

### 2.1 処理フロー
1. LLMが要件テキストから品目・数量・単価を構造化
2. コードノードで小計・消費税・合計を計算
3. テンプレート変換で見積書フォーマットに整形

### 2.2 出力フォーマット
```
━━━━━━━━━━━━━━━━━━━━━━━━━
        御見積書
━━━━━━━━━━━━━━━━━━━━━━━━━
宛先：○○株式会社 御中
件名：○○に関する御見積

| No. | 品目 | 数量 | 単位 | 単価（税抜） | 金額 |
|-----|------|------|------|------------|------|
| 1   | ...  | ...  | ...  | ...        | ...  |

小計：¥XXX,XXX
消費税（10%）：¥XX,XXX
合計：¥XXX,XXX

有効期限：発行日より30日間
備考：
```

## 3. ワークフロー構成

```
開始 → LLM（要件構造化）→ コード（金額計算）→ テンプレート変換（見積書フォーマット）→ 回答
```

## 4. モデル設定

| 項目 | 設定値 |
|------|--------|
| **モデル** | gemini-2.5-flash |
| **Temperature** | 0.2 |
| **メモリ** | 有効 / ウィンドウサイズ10 |
