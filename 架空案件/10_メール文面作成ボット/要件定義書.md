# 要件定義書：メール文面作成ボット

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | メール文面作成ボット |
| **Difyアプリ種別** | チャットフロー（advanced-chat） |
| **対象ユーザー** | 全社員（特に営業・カスタマーサポート） |
| **解決する業務課題** | ビジネスメールの文面を毎回1から考える手間、トーンや敬語レベルの不統一 |

## 2. 機能要件

### 2.1 メールタイプ
| タイプ | 説明 |
|--------|------|
| お礼メール | 打ち合わせ後、受注後、面談後 |
| 依頼メール | 見積依頼、資料送付依頼、確認依頼 |
| 謝罪メール | 納期遅延、不具合報告、対応遅延 |
| 報告メール | 進捗報告、完了報告、問題報告 |
| 案内メール | 会議案内、イベント案内、変更通知 |
| フォローアップ | 未返信へのリマインド、追加情報提供 |

### 2.2 トーン設定
- **フォーマル**: 社外・上位者向け（最上級の敬語）
- **スタンダード**: 社外一般（標準ビジネス敬語）
- **セミカジュアル**: 社内・親しい取引先

### 2.3 出力
- 件名
- 本文（宛名・挨拶・本題・結び・署名欄）
- ※メモリにより修正指示に対応

## 3. ワークフロー構成

```
開始 → LLM（メール生成）→ 回答
```

## 4. モデル設定

| 項目 | 設定値 |
|------|--------|
| **モデル** | gemini-2.5-flash |
| **Temperature** | 0.4（少し創造性を許容） |
| **メモリ** | 有効 / ウィンドウサイズ10 |
